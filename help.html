CMv3 API Help

The API Request form is a simplification of the Lumen CMv3 API process, allowing users to gather information quickly. More features will be added as the API expands. The output is JSON formatted and readily styled for a copy-paste job back into the CMv3 platform.

All requests require an API Key and Secret combination. The API Key and Secret must be accessible to a CMv3 SubID.

All requests require at least a SubID to obtain a response. Requests that only need SubID:

  • Origin Shield IPs
  • Edge IPs
  • Subscriber Current Configuration
  • Subscriber Candidate Configuration
  • Deleted Configuration IDs
  • Recent Transactions

Some requests require other parameters such as Transaction ID or Config Name:

  • Display Deleted Configuration: This requires the Configuration name obtained from the 'Deleted Configuration ID'
  • Transaction ID Details: This transaction ID is obtained from 'Recent Transactions'
